DNS Spoofing (GFW)

DNS is the first step in any connection: the browser must resolve a domain name to an IP address before it can connect. Standard DNS queries travel over UDP port 53 in plaintext, which makes them visible to any device on the network path.

A middlebox such as the Great Firewall of China (GFW) can inspect these queries in real time. When it detects a query for a censored domain, it races to inject a spoofed response containing a bogus IP address. The real DNS response from the legitimate server still arrives — but later. Since the client typically uses whichever response arrives first, the spoofed answer wins and the connection fails because the resolved IP address doesn't lead to the real server.

Before enabling any censorship, let's observe a normal DNS lookup. Type dig +short www.example.com in the terminal below and look at the response: you should see the correct IP address for the domain. Use the packet viewer to inspect the DNS query and response on the wire.

2b. Connection Fails

With censorship still active, try curl http://www.example.com/. The connection will fail because the resolved IP address is bogus and there is no server listening at that address.

3. Bypassing DNS Censorship

If we already know the correct IP address, we can bypass DNS entirely. Run curl --resolve www.example.com:80:104.18.26.120 http://www.example.com/ to tell curl to skip the DNS lookup and connect directly to the real server. The connection should succeed despite censorship still being active.
